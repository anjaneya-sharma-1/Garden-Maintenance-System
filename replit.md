# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Key Commands

- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally

See the `pnpm-workspace` skill for workspace structure, TypeScript setup, and package details.

## Garden Maintenance System (`artifacts/garden-app` + `artifacts/api-server`)

A full-stack web app for home gardeners.

**Features**: cookie-session auth (signup, login, logout), profile editing, plant catalog & knowledge search, personal garden CRUD, AI-style care routine generation, simulated soil readings (pH, temp, humidity, moisture) with 24h history, watering logging + 30-day report, reminders (with optional repeat), dashboard with health/activity, admin panel (users + catalog CRUD + stats).

**Frontend**: React + Vite + Tailwind + shadcn/ui + react-query + wouter + recharts + react-hook-form + zod. All data hooks are imported from `@workspace/api-client-react` (auto-generated by Orval).

**Backend**: Express 5 + Drizzle + Postgres. Auth uses scrypt-hashed passwords + DB-backed session table with an `gms_sid` cookie. Soil readings are deterministically simulated (no soil_readings table) from `gardenPlantId + timestamp` so charts are stable across requests.

**Seeding**: `cd artifacts/api-server && pnpm exec esbuild src/seed.ts --bundle --platform=node --format=cjs --outfile=dist/seed.cjs && node dist/seed.cjs`. Idempotent. Creates 10 catalog plants, an admin (`admin@gms.local` / `gardener123`) and a demo user (`demo@gms.local` / `gardener123`) with plants, watering logs, and reminders.

**Plant images**: stored as static assets at `artifacts/garden-app/public/plants/*.png`, referenced by relative URL in catalog rows.

**No emojis** are used in the UI.
